# Components

> React components for FDDHub and FDDAdvisor platforms

## Overview

This directory contains all React components used across the application. Components are organized by feature area, with shared UI primitives in the `ui/` subdirectory.

## Directory Structure

```
components/
├── ui/                     # shadcn/ui primitives (auto-generated)
├── admin/                  # Admin-only components
├── dashboard/              # Dashboard view components
├── fdd-*.tsx              # FDD-related components
├── franchise-*.tsx        # Franchise display components
├── *-modal.tsx            # Modal dialogs
└── *.tsx                  # Feature components
```

## Component Patterns

### Client Components
Most components are client components (`"use client"`) for interactivity:

```tsx
"use client"

import { useState } from "react"

export function MyComponent() {
  const [state, setState] = useState()
  // ...
}
```

### Props Pattern
Components use TypeScript interfaces:

```tsx
interface MyComponentProps {
  data: SomeType
  onAction?: (id: string) => void
  className?: string
}

export function MyComponent({ data, onAction, className }: MyComponentProps) {
  // ...
}
```

### Modal Pattern
Modals use shadcn Dialog:

```tsx
import { Dialog, DialogContent, DialogHeader, DialogTitle } from "@/components/ui/dialog"

export function MyModal({ open, onOpenChange }: { open: boolean; onOpenChange: (open: boolean) => void }) {
  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Title</DialogTitle>
        </DialogHeader>
        {/* Content */}
      </DialogContent>
    </Dialog>
  )
}
```

## Styling

- **Tailwind CSS v4** for all styling
- **CSS Variables** for theming (see `globals.css`)
- **cn()** utility for conditional classes

```tsx
import { cn } from "@/lib/utils"

<div className={cn("base-class", isActive && "active-class")} />
```

## State Management

- **React hooks** for local state
- **URL params** for shareable state
- **Supabase Realtime** for live updates (future)

No global state library - each component manages its own state.
